# Troubleshooting - Gemini Remote Bridge

This document provides solutions for common issues when setting up remote connections across different platforms.

## 1. SSH Key Rejected
If the bridge indicates that the key was rejected after automatic injection attempts, authorize it manually from your Local Mac:

```bash
ssh-copy-id -i ~/.ssh/id_rsa_bridge.pub [user]@[remote-ip]
```

## 2. SFTP / Mutagen Error (Routers & Embedded)
If Mutagen fails with "unable to copy agent binary" or "sftp-server not found", the remote device lacks an SFTP server.

### For OpenWrt:
```bash
opkg update && opkg install openssh-sftp-server
```

### For Entware (ASUS, Padavan, etc.):
```bash
opkg update && opkg install sftp-server
```

## 3. Linux Server Setup
Ensure proper permissions for the SSH directory on the remote Linux machine:

```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
touch ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys
```

## 4. macOS Remote Login
To connect to a remote Mac, you must enable **Remote Login** in **System Settings > General > Sharing**. Make sure your user is added to the allowed users list.

## 5. SSH-Only Compatibility Mode
If the device is incompatible with Mutagen's agent (unsupported architecture or missing libraries), the bridge will automatically activate **SSH-Only Mode**.
- Commands will execute remotely.
- Real-time file synchronization will be disabled.
- Gemini will use standard shell commands to manage files directly.
