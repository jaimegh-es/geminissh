🆘 Connection Troubleshooting Guide

This guide helps you resolve common connection and synchronization issues across different platforms.

All Platforms 1. SSH Key Rejected

If the bridge reports that the SSH key was rejected, you need to manually authorize your Mac's bridge key on the remote machine.

Run this command on your Local Mac Terminal:

ssh-copy-id -i ~/.ssh/id_rsa_bridge.pub [user]@[remote-ip]

Routers / Embedded 2. SFTP / Mutagen Error

If you see sftp-server not found or agent installation failures, the remote device lacks an SFTP server, which Mutagen prefers.

For OpenWrt Routers:

opkg update && opkg install openssh-sftp-server

For Entware (ASUS, Padavan, etc.):

opkg update && opkg install sftp-server

Linux Servers 3. Permission & Path Issues

Ensure the remote user has write permissions to the destination directory and that the .ssh folder has correct permissions.

Run these commands on the Remote Linux machine:

mkdir -p ~/.ssh && chmod 700 ~/.ssh
touch ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys

macOS Remote 4. Remote Login Setup

To connect to another Mac, you must enable Remote Login and ensure the user is in the allowed list.

On the Remote Mac:

  • Go to System Settings > General > Sharing
  • Enable Remote Login
  • Click the (i) icon and ensure your user has access.

Fallback 5. SSH-Only Mode

If Mutagen is incompatible with the remote architecture (e.g., very old ARM/MIPS or missing glibc), the bridge will automatically switch to Direct Command Mode. File syncing will be disabled, but Gemini will still be able to execute commands and manage files via SSH.
